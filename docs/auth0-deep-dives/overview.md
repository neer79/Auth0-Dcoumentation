# Auth0 deep dives overview

This section provides deeper Auth0-specific playbooks for architecture, business use cases, integration patterns, onboarding, extensibility, authentication methods, communications, cryptographic operations, and ongoing tenant support.

Use these pages when the short feature descriptions are not enough and a team needs to design, implement, operate, or support a real Auth0 capability.

## Deep-dive map

| Area | Use when |
| --- | --- |
| Business use cases | You need to map Auth0 capabilities to workforce, customer, partner, SaaS, or machine identity scenarios |
| Reference architecture | You need an end-to-end Auth0 platform architecture with runtime, control, and operations planes |
| Integration patterns | Application teams need approved OIDC, OAuth, SAML, API, and service integration patterns |
| Application onboarding | Teams need a repeatable intake-to-production process for new applications |
| Customer IdP onboarding | B2B customers need to bring Azure AD, Okta, ADFS, Google Workspace, or SAML/OIDC providers |
| Actions patterns | Teams need examples of where Actions fit and how to govern them |
| MFA methods | Security teams need factor-by-factor guidance and rollout decisions |
| Passwordless | Product teams need passwordless options, risks, and implementation decisions |
| Email, SMS, and error pages | Teams need to customize user-facing communications and error experiences |
| Key rotation | Operations teams need signing key, secret, certificate, and token-impact procedures |
| Tenant setup and support | Platform teams need initial setup and ongoing support runbooks |

## Recommended sequence

1. Start with [Business Use Cases](business-use-cases.md).
2. Review [Reference Architecture](reference-architecture.md).
3. Select patterns from [Integration Patterns](integration-patterns.md).
4. Use [Application Onboarding Solution](application-onboarding-solution.md) for application teams.
